Online Music Recommendation Toll

Abstract

This site represents an online tool dedicated to the discovery of various music related information. The site is based on the premise that the song is the basis of all music and thus the aim of this tool is to search for new songs based on user given preferences and to give new insights to the user about the record's authors, genera, type, history and associations.

User flow

From a user's standpoint he or she will be able to do a number of actions, each one with an associated page. The following sub chapters will be placed in order and describe a general user flow.

1. Login\Register

To gain access to any utility from the music recommendation tool a user must first register or login to the site. To do this the user will be prompted to enter his or hers email and password. If the authentication succeeds the user will remain logged in until a certain time expires or the logout option is pressed. In the logged in state the user will now be granted access to the tool's utilities.

2. Playlist Management

A logged in user will now be able to see his/hers available playlists. On this page they can create a new playlist, delete a playlist or select a playlist to begin the process of searching for new songs to add to the respective list or they can chose to remove songs from it. Another option that is now available to the user on this page is the ability to share a selected playlist with friends via an online social platform.

3. Music Searching\Playlist Editing

On this page the user will be presented with a text-box were he\she will be able to type a request in normal natural language (preferably in english) in which various statements and suggestions can be made regarding the songs and musics they want to receive. These requests will be then sent to the server which will then process them and provide the user with a list of songs accompanied by various information regarding them, like their artists, history genera and more. A user can then chose to add these songs to their playlist or they can chose to discard the results and send over a new request.
